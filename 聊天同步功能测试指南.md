# 聊天同步功能测试指南

## 问题修复说明

### 原问题
- WebSocket 成功接收消息并触发 `offergod:chat:message` 事件
- 但没有代码监听该事件并保存到 chat store
- 导致"发现一个沟通都没有"

### 修复内容
在 `src/entrypoints/content.ts` 中添加了两个事件监听器：

1. **`offergod:chat:message` 监听器**
   - 接收 WebSocket 消息事件
   - 自动创建或更新会话（chatSessions）
   - 保存消息到对应会话的 messages 数组
   - 更新会话的最后消息和时间
   - 计算未读消息数量

2. **`offergod:chat:read` 监听器**
   - 接收已读消息事件
   - 标记消息状态为 'read'

### 数据存储结构
```javascript
// chrome.storage.local.chatSessions
[
  {
    id: "jobId_bossUid",
    jobId: "岗位ID",
    bossUid: "Boss用户ID",
    bossName: "Boss姓名",
    companyName: "公司名称",
    jobTitle: "岗位名称",
    lastMessage: "最后一条消息内容",
    lastMessageTime: 时间戳,
    unreadCount: 未读数量,
    messages: [
      {
        id: "消息ID",
        sessionId: "会话ID",
        role: "user" | "boss",
        content: "消息内容",
        time: 时间戳,
        status: "sent" | "read"
      }
    ]
  }
]
```

### ❌ 失败情况处理

#### 情况 1：WebSocket 未连接
**症状**：没有看到"WebSocket 已连接"日志

**原因**：
- 用户信息获取失败
- WebSocket 连接被阻止

**解决**：
1. 刷新页面重试
2. 检查网络连接
3. 查看控制台是否有错误信息

#### 情况 2：收到消息但未保存
**症状**：看到"收到聊天消息事件"但没有"聊天消息已保存"

**原因**：
- 保存过程出错
- 数据格式不正确

**解决**：
1. 查看控制台错误日志
2. 检查 userInfo 是否存在
3. 执行 `chrome.storage.local.get('userInfo', console.log)` 验证

#### 情况 3：事件未触发
**症状**：发送消息后没有任何日志

**原因**：
- WebSocket 未正确触发事件
- 事件监听器未注册

**解决**：
1. 确认扩展已重新加载
2. 检查 content.js 是否正确编译
3. 查看 `.output/chrome-mv3/content-scripts/content.js` 是否包含事件监听代码

## 代码变更摘要

### 修改文件
- `src/entrypoints/content.ts` (+95 行)

### 新增功能
1. 监听 `offergod:chat:message` 事件
2. 监听 `offergod:chat:read` 事件
3. 自动创建会话记录
4. 保存消息到 chrome.storage.local
5. 计算未读消息数量
6. 更新会话最后消息信息

### 数据流程
```
WebSocket 接收消息
  ↓
useWebSocket 解码 protobuf
  ↓
触发 window.dispatchEvent('offergod:chat:message')
  ↓
content.ts 监听器接收事件
  ↓
处理消息数据（创建/更新会话）
  ↓
保存到 chrome.storage.local.chatSessions
  ↓
扩展面板从 storage 读取并显示
```
